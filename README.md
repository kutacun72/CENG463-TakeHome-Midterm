## Project Overview

This repository contains the Python scripts, generated figures, result tables, and final PDF report for the CENG 463 Machine Learning take-home midterm. The project consists of five tasks: regression, imbalanced classification, dimensionality reduction, clustering, and neural network-based image classification. The final report is submitted as a compiled PDF, and the LaTeX source file is not included in this repository.

## Folder Structure

- `Q1/`: Regression experiments
- `Q2/`: Imbalanced classification experiments
- `Q3/`: Dimensionality reduction experiments
- `Q4/`: Clustering experiments
- `Q5/`: Neural network, transfer learning, interpretability, and adversarial robustness experiments
- `report/`: Final PDF report
- `README.md`: Project explanation and run instructions

## How to Run the Scripts

Open a terminal in the project root folder and run the scripts inside each question folder. For Question 1, run `cd Q1` and then `python main.py`. For Question 2, run `cd Q2` and then `python main.py`. For Question 3, run `cd Q3` and then `python q3_dimensionality_reduction.py`. For Question 4, run `cd Q4` and then `python q4_clustering.py`. For Question 5, run `cd Q5` and then run `python q5_neural_networks.py`, `python q5_transfer_learning.py`, and `python q5_interpretability_robustness.py`.

Question 5 is split into three scripts. `q5_neural_networks.py` trains and evaluates the MLP and CNN models. `q5_transfer_learning.py` trains and evaluates the MobileNetV2 transfer learning model. `q5_interpretability_robustness.py` generates misclassified examples, Grad-CAM visualisations, and FGSM adversarial robustness results.

## Important Note for Question 5: CIFAR-10 Dataset

Question 5 uses the CIFAR-10 dataset. Large dataset files are not included in this repository to keep the repository size manageable. The excluded files and folders include `cifar10_data/`, `cifar-10-python.tar.gz`, trained `.keras` model files, extracted dataset folders, Python cache files, and virtual environment folders.

If you want to run the Q5 scripts from scratch, make sure that the CIFAR-10 Python archive is available locally. The required file is `cifar-10-python.tar.gz`, which can be downloaded from the official CIFAR-10 dataset page: https://www.cs.toronto.edu/~kriz/cifar.html

After downloading, place the file in the Keras dataset cache directory before running the Q5 scripts. On Windows, this is usually `C:\Users\<YOUR_USERNAME>\.keras\datasets\`, so the file should be located as `C:\Users\<YOUR_USERNAME>\.keras\datasets\cifar-10-python.tar.gz`. If the path is different on another computer, update the CIFAR-10 archive path inside the Q5 scripts before running them.

The trained neural network model files are also not included in this repository. The excluded model files include `q5_mlp_model.keras`, `q5_cnn_model.keras`, and `q5_transfer_model.keras`. These files are generated automatically when the Q5 scripts are run. To reproduce all Q5 outputs, run the scripts in this order: first `q5_neural_networks.py`, second `q5_transfer_learning.py`, and third `q5_interpretability_robustness.py`. The third script depends on the trained model files generated by the first two scripts.
